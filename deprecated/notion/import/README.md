# Notion CSV 导入说明

这套文件用于快速把 Profit 项目管理数据导入 Notion。

## 文件

- `需求池.csv`
- `版本日志.csv`
- `字段库.csv`
- `规则库.csv`

## 导入步骤

1. 打开 Notion。
2. 左侧点击 `Import`。
3. 选择 `CSV`。
4. 依次导入 4 个 CSV 文件。
5. 导入后，把生成的 4 个数据库移动到同一个页面，页面命名为 `Profit 项目管理`。

## 导入后建议调整字段类型

Notion 从 CSV 导入后，大多数字段会是 Text，需要手动改成更合适的类型。

### 需求池

- `状态` 改为 Select：待处理 / 进行中 / 已完成 / 暂缓
- `优先级` 改为 Select：高 / 中 / 低
- `模块` 改为 Select：商品页 / 配置页 / 规则编辑器 / 更新发布 / Excel配置
- `完成时间` 改为 Date
- `关联版本` 可改为 Relation → 版本日志

### 版本日志

- `发布时间` 改为 Date
- `状态` 改为 Select：计划中 / 已发布 / 回滚
- `验证状态` 改为 Select：未验证 / 通过 / 有问题
- `关联需求` 可改为 Relation → 需求池

### 字段库

- `层级` 改为 Select：平台级 / 商品级 / SKU级
- `输入输出` 改为 Select：输入 / 输出
- `类型` 改为 Select：文本 / 数字 / 下拉 / 布尔 / 日期

### 规则库

- `规则排序` 改为 Number
- `启用` 改为 Checkbox
- `规则类型` 改为 Select：查表 / 映射 / 计算 / 条件 / 输出
- `输入字段` 可改为 Relation → 字段库
- `输出字段` 可改为 Relation → 字段库
- `关联需求` 可改为 Relation → 需求池

## 注意

- CSV 不能自动创建 Relation，需要导入后手动把 Text 字段转换为 Relation。
- `输入字段` 和 `输出字段` 目前用分号 `;` 分隔，便于你手动关联字段库条目。
- 如果只是想记录项目，不建立 Relation 也可以直接使用。

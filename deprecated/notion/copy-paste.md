# Profit 项目管理

## 使用步骤

1. Notion 新建页面，命名为 `Profit 项目管理`
2. 按下方说明创建 4 个内嵌数据库（输入 `/database` → `Table - Inline`）
3. 配置属性 → 创建模板 → 录入初始数据

---

## 需求池

创建方式：`/database` → `Table - Inline` → 命名为 `需求池`

### 属性

- 名称（Title）、状态（Select：待处理/进行中/已完成/暂缓）、优先级（Select：高/中/低）
- 模块（Select：商品页/配置页/规则编辑器/更新发布/Excel配置）
- 需求描述（Text）、目标效果（Text）、验收标准（Text）
- 关联版本（Relation → 版本日志）、创建时间（Created time）、完成时间（Date）

```

- [ ] 
```

### 数据（6 条，状态均为“已完成”，关联版本 1.5.1）

**1. 计算节点改为可视化公式构造器**（高优先级 / 规则编辑器）
- 目标效果：通过按钮添加字段、节点、常量、运算符和括号，自动拼公式
- 验收：不再有手写表达式入口 / + - × ÷ 括号可用 / 5 条规则已迁移

**2. 新建商品页三段横向布局**（高优先级 / 商品页）
- 目标效果：平台级、商品级、SKU级三段横向铺开
- 验收：商品ID名称在商品级 / 国家平台模板在平台级 / SKU前缀变体在SKU级

**3. SKU 变体属性改为小表格**（高优先级 / 商品页）
- 目标效果：属性值逐个小输入框，不再手写 | 分隔符
- 验收：+ 值可新增 / × 可删单个值 / 变体变化后自动刷新 SKU

**4. 新增平台级字段层级**（中优先级 / 配置页）
- 目标效果：字段编辑可选平台级，平台级字段在新建商品时正确渲染
- 验收：FIELD_LEVELS 含平台级 / 字段编辑可选 / 渲染正常

**5. 更新弹窗日志支持换行**（低优先级 / 更新发布）
- 目标效果：更新弹窗多行显示日志
- 验收：多行显示 / 长日志可滚动

**6. 计算配置改为规则级 DAG 流程**（高优先级 / 规则编辑器）
- 目标效果：一个模板多个规则，各有独立 DAG
- 验收：Excel 一行一个规则 / 导入导出正常

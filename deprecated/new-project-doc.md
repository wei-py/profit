# 利润工具 v2 - 新项目文档

## 项目概述

**利润工具** 是一个面向跨境电商卖家的桌面端利润计算工具。核心功能：选择国家平台 → 选择计算模板 → 填入商品参数 → 自动计算全部费用（佣金、运费、手续费等） → 保存商品记录。

技术方案：**Excel 驱动配置**，所有业务规则、字段定义、选项、费率数据均存储在 `.xlsx` 文件中，程序代码作为纯粹的读取器和计算器。

### 当前版本问题（v1）

v1 数据模型共 14 个 sheet，存在以下痛点：

- 条件树嵌套（`parent_group_id`）在 Excel 中不可见不可编辑
- 规则动作使用 JSON 配置（`config_json`），非开发者无法理解
- 理解一条费用规则需要跨 5-6 个 sheet 追溯
- `presets` + `preset_params` 是冗余抽象层，与 `fields` 大量重复
- 国家平台本应是核心实体，却被边缘化

### v2 设计理念

1. **扁平化**：无树形嵌套，无 JSON 配置，一行规则完全自描述
2. **领域化**：用业务语言（佣金、运费、手续费）替代通用术语（规则、条件、动作）
3. **Excel 友好**：所有配置在单元格中可见可编辑，不需要理解代码
4. **国家为核心**：国家+平台组合是最大、最中心的锚点表

| 货币符号 | 文本 | 显示符号     | R$                     |

| 启用     | 布尔 | TRUE/FALSE   | TRUE                   |
| 排序     | 数字 | 显示顺序     | 1                      |

| **所属国家平台** | 文本 | **外键→国家平台.编号**                        | cp_br              |

每个模板是一个具名的费用计算方案，关联一个国家平台。取代旧版的 presets + rule_sets 双层结构。

| 列名         | 类型 | 说明             | 示例                     |
| ------------ | ---- | ---------------- | ------------------------ |
| 编号         | 文本 | 唯一标识         | tpl_br_default           |
| 名称         | 文本 | 面向用户的名称   | 巴西通用模板             |
| 所属国家平台 | 文本 | 关联国家平台编号 | cp_br                    |
| 启用         | 布尔 | TRUE/FALSE       | TRUE                     |
| 说明         | 文本 | 模板用途描述     | 巴西ML站默认费用计算模板 |

### Sheet 6: 费用规则（最大最详细的表，约 50-500 行）⭐

整个工作簿的核心。**每行是一条完整的费用计算规则，所有细节都在同一行中可见，无 JSON 配置，无树形嵌套。**

条件组合规则：

- **AND**：同一行的条件1+条件2 全部满足才触发
- **OR**：拆成两条规则，相同的输出字段键和计算顺序
- **IF-ELSE**：两条规则，一条条件=X，一条条件≠X

| 列名               | 类型 | 说明                                           | 示例                            |
| ------------------ | ---- | ---------------------------------------------- | ------------------------------- |
| 编号               | 文本 | 唯一标识                                       | r_br_001                        |
| 所属模板           | 文本 | 关联模板编号                                   | tpl_br_default                  |
| **输出字段键**     | 文本 | **写入结果时用的字段键（中文），对应计算字段** | 销售佣金费率                    |
| 费用名称           | 文本 | 业务名称                                       | 销售佣金费率                    |
| 计算顺序           | 数字 | 越小越先执行                                   | 10                              |
| 启用               | 布尔 | 是/否                                          | 是                              |
| 条件1字段          | 文本 | 条件1: 判断的字段键                            | 是否包邮                        |
| 条件1运算符        | 选项 | 等于/不等于/大于/大于等于/小于/小于等于        | 等于                            |
| 条件1值            | 文本 | 比较值                                         | 是                              |
| 条件1值2           | 文本 | between 的上界（暂不常用）                     | —                               |
| 条件2字段          | 文本 | 条件2: AND 附加条件（可选）                    | —                               |
| 条件2运算符        | 选项 | 同上                                           | —                               |
| 条件2值            | 文本 | —                                              | —                               |
| 条件2值2           | 文本 | —                                              | —                               |
| **计算方式**       | 选项 | **查表 / 百分比 / 固定值 / 加总 / 公式**       | 查表                            |
| 查表名称           | 文本 | 费率表 Sheet 名称（查表用）                    | commission_table                |
| 匹配方式           | 选项 | 精确 / 区间                                    | 精确                            |
| 输入映射           | 文本 | `字段键=表列名`，逗号分隔                      | 刊登类型=刊登类型,商品类目=类目 |
| 输出列             | 文本 | 费率表中的结果列名                             | 费率                            |
| 百分比基数         | 文本 | 百分比方式的计算基数字段                       | 售价                            |
| 百分比值           | 数字 | 固定百分比（如 4.8 = 4.8%）                    | 4.8                             |
| **百分比来源字段** | 文本 | **动态百分比：引用另一个结果字段键**           | 销售佣金费率                    |
| 固定金额           | 数字 | 固定值方式的固定金额                           | 0                               |
| **加总字段**       | 文本 | **加总方式：逗号分隔的待加总字段键**           | 销售佣金金额,运费,支付手续费    |
| **公式**           | 文本 | **公式方式：中文字段键的算术表达式**           | 售价 - 总费用 - 成本价          |
| 累加               | 布尔 | 是=追加到已有值, 否=覆盖                       | 否                              |
| 说明               | 文本 | 规则说明                                       | —                               |

> **百分比值 vs 百分比来源字段**：二选一。百分比值是固定数字，百分比来源字段是引用前面规则的输出字段键。
>
> **公式**直接用中文字段键写表达式，如 `售价 - 总费用 - 成本价`、`净利润 / 售价`。引擎执行时把字段键替换为数值后求值。

### Sheet 7: 模板参数（可选，默认值，约 10-100 行）

为模板中的输入字段提供默认值。如不填此表，系统使用计算字段中的全局属性显示所有字段。

| 必填     | 布尔 | 覆盖全局必填     | TRUE           |

独立的查表数据 sheet，由费用规则的"查表名称"列引用。列结构完全由用户定义。

## 实体关系图

```
国家平台 (cp_id)  ← 根锚点，所有数据向上归属
  │
  ├──< 计算字段 (cp_id)         每个国家平台有自己的一套字段定义
  │     │
  │     └──> 选项组 (cp_id)     字段中的 select 类型引用选项组
  │             │
  │             └──< 选项值 (group_id)  选项明细，通过选项组间接归属国家
  │
  ├──< 选项组 (cp_id)          每个国家平台有自己的一套选项
  │     │
  │     └──< 选项值 (group_id)
  │
  ├──< 计算模板 (cp_id)         每个国家平台下可建多个模板
  │     │
  │     ├──< 费用规则 (template_id)   模板下的计费规则，按顺序执行
  │     │     │
  │     │     ├─ 条件字段   → 计算字段
  │     │     ├─ 输入映射   → 计算字段
  │     │     ├─ 百分比基数 → 计算字段
  │     │     └─ 查表名称   → [费率表 Sheet]
  │     │
  │     └──< 模板参数 (template_id)   可选默认值，引用计算字段
  │
  └── [费率表 Sheet]              通过费用规则.查表名称间接引用
```

**关键约束：**

- 字段的选项组必须是同国家平台下的选项组（跨国家引用报错）
- 模板只能引用同国家平台下的字段（跨国家引用无意义）
- 所有数据最终可追溯到唯一的国家平台

## 多 SKU 计算流程

### 字段的两层体系

`计算字段.层级` 将字段分为两大类：

```
┌─────────────────────────────────────┐
│  Product 级（商品级，所有 SKU 共享）  │
│  刊登类型、类目、是否包邮            │
│  用户填一次，所有 SKU 自动继承       │
└─────────────────────────────────────┘
              │
              │ 合并
              ↓
┌─────────────────────────────────────┐
│  SKU 级（每个变体独立）              │
│  售价、成本价、重量、图片            │
│  不同颜色/尺码填不同值              │
└─────────────────────────────────────┘
```

> **图片为什么是 SKU 级？** 红色SKU的商品图是红色的，蓝色SKU是蓝色的。商品级"共享图片"没有实际意义，每个变体独立管理自己的图片。

### 计算流程图

```
用户操作                        费引擎内部
────────                      ────────────

[选择国家平台] → cp_br
  └─ 模板下拉框只显示 cp_br 下的模板

[选择模板] → tpl_br_default
  ├─ 加载该模板的 Product级和SKU级字段（WHERE cp_id='cp_br'）

[填写商品信息]
  ├─ Product级字段: 刊登类型=premium, 类目=美容, 包邮=否
  └─ (所有SKU共享这些值)

[添加变体属性] 颜色:红,蓝  尺码:S,M
  └─ 笛卡尔积 → 4个SKU组合

[填写SKU表格] (列完全由 计算字段.层级=sku 决定，不写死)
  示例：当前巴西模板中 SKU级字段 = {售价, 成本价, 重量, 款号, 图片}
  实际列 = 所有输入类SKU字段 + 图片 + 所有输出类SKU字段（结果列）

  ┌────────┬────────┬────────┬────────┬────────┬──────────┬──────────┬──────────┐
  │ SKU    │ 款号   │ 售价   │ 成本价 │ 重量   │ 图片     │ 佣金     │ 运费     │...
  ├────────┼────────┼────────┼────────┼────────┼──────────┼──────────┼──────────┤
  │ 红-S   │ RS001  │  99.00 │  25.00 │ 0.30   │ red_S_1  │  15.84   │  12.35   │
  │ 红-M   │ RM001  │ 109.00 │  28.00 │ 0.35   │ red_M_1  │  17.44   │  15.45   │
  │ 蓝-S   │ BS001  │  89.00 │  24.00 │ 0.30   │ blue_S_1 │  14.24   │  12.35   │
  │ 蓝-M   │ BM001  │ 105.00 │  27.00 │ 0.35   │ blue_M_1 │  16.80   │  15.45   │
  └────────┴────────┴────────┴────────┴────────┴──────────┴──────────┴──────────┘

  输入列（可编辑）= 层级=sku + 输入输出=input 的字段
  输出列（只读）  = 层级=sku + 输入输出=output 的字段
  图片列         = 始终存在，每SKU独立挂图

[计算] ──→ 逐 SKU 合并输入，调用引擎

  SKU: 红-S
  合并输入 = {刊登类型:premium, 商品类目:beauty, 是否包邮:否,  ← 商品级
             售价:99, 成本价:25, 重量:0.30}                     ← SKU级
     │
     ├─ 规则1 查表: premium+beauty → 佣金费率 0.16
     ├─ 规则2 百分比: 99 × 0.16 → 佣金金额 15.84
     ├─ 规则3 固定值: 是否包邮≠是 → 跳过
     ├─ 规则4 查表: 售价99∈[79,99.99], 重量0.30∈[0,0.3] → 运费 12.35
     ├─ 规则5 百分比: 99 × 4.8% → 手续费 4.75
     ├─ 规则6 加总: 15.84+12.35+4.75 = 总费用 32.94
     ├─ 规则7 公式: 99-32.94-25 = 净利润 41.06
     └─ 规则8 公式: 41.06/99 = 利润率 41.47%

  SKU: 红-M
  合并输入 = {刊登类型:premium, 商品类目:beauty, 是否包邮:否,
             售价:109, 成本价:28, 重量:0.35}
     │
     ├─ 规则1 查表: 同条件 → 佣金费率 0.16
     ├─ 规则2 百分比: 109 × 0.16 → 佣金金额 17.44
     ├─ 规则4 查表: 售价109∈[100,119.99], 重量0.35∈[0.3,0.5] → 运费 15.45
     ├─ 规则5 百分比: 109 × 4.8% → 手续费 5.23
     ├─ 规则6 加总: 17.44+15.45+5.23 = 总费用 38.12
     ├─ 规则7 公式: 109-38.12-28 = 净利润 42.88
     └─ 规则8 公式: 42.88/109 = 利润率 39.34%

  ... 蓝-S、蓝-M 同理计算 ...

[保存] ──→ 一个商品下 4 个 SKU，每个 SKU 独立计算结果
```

### 为什么不需要在费用规则里区分层级？

费用规则不关心字段是 product 级还是 SKU 级。引擎传入的是**已合并完成的输入**：

```javascript
// 每个 SKU 独立计算时
mergedInputs = { ...所有Product级字段值, ...该SKU的SKU级字段值 };
result = execute(feeRules, lookupTables, mergedInputs);
```

模板（费用规则）只管"条件是什么 → 算什么"，不管数据从哪个层级来。

### 商品记录约束

**唯一性：** `商品ID + SKU码` 在工作簿内唯一。同一商品ID下不允许重复 SKU 码。

**一致性：** 程序读取时校验同一商品ID下的商品级字段必须一致。如不一致（例如 P001 红-S 类目=beauty、P001 红-M 类目=fashion），提示用户修正，不静默取第一行。

**输出列名：** 输出列名直接使用计算字段的字段键（如 `销售佣金金额`），不简写（如 `佣金`），零映射。

**计算结果为缓存：** 输出列保存的是计算时的快照。配置（佣金率、运费表）变更后，旧结果不会自动更新。程序提供"按当前规则重算全部"功能，用户也可在 Excel 中直接改输入列后重新加载计算。

### 变体属性列

变体属性不作为文本合并存储，而是动态拆成独立列。用户定义"颜色:红,蓝"和"尺码:S,M"时，商品记录自动生成 `颜色` 列和 `尺码` 列。属性名由用户在 CreatePanel 中定义，不同模板可有不同变体属性。

## 计算引擎核心逻辑

```
function execute(费用规则列表, 字段定义, 费率表集合, 用户输入):
  results = {}
  sorted = 费用规则.filter(启用=true).sort(计算顺序升序)

  for each rule in sorted:
    # 1. 条件判断
    if rule.条件1字段 != "":
      if NOT match(inputs[rule.条件1字段], rule.条件1运算符, rule.条件1值, rule.条件1值2):
        continue  # 条件不满足，跳过
    if rule.条件2字段 != "":
      if NOT match(inputs[rule.条件2字段], rule.条件2运算符, rule.条件2值, rule.条件2值2):
        continue

    key = rule.输出字段键

    # 2. 按计算方式执行
    switch rule.计算方式:
      case "查表":
        // 精确匹配：所有输入映射列值相等
        // 区间匹配：[下限, 上限] 闭区间，多行命中取表顺序第一条
        row = lookupTable(rule.查表名称, rule.输入映射, rule.匹配方式, inputs, results)
        val = Number(row[rule.输出列])
        results[key] = val

      case "百分比":
        base = results[rule.百分比基数] ?? inputs[rule.百分比基数] ?? 0
        if rule.百分比来源字段 != "":
          rate = results[rule.百分比来源字段]  # 动态：引用另一个结果
        else:
          rate = rule.百分比值 / 100            # 固定：如 4.8 → 0.048
        results[key] = base * rate

      case "固定值":
        results[key] = rule.固定金额

      case "加总":
        fields = rule.加总字段.split(',')
        results[key] = sum(fields.map(f => results[f] ?? inputs[f] ?? 0))

      case "公式":
        // 将中文字段键替换为数值，仅允许 + - * / ( ) 和数字
        expr = rule.公式
        // 按字段键长度从长到短替换，避免短名误替换长名
        const sortedKeys = allFieldKeys.sort((a,b) => b.length - a.length)
        for (const fk of sortedKeys) {
          const val = results[fk] ?? inputs[fk] ?? 0
          expr = expr.replaceAll(fk, Number(val))
        }
        // 安全校验：替换后只含数字、运算符、空格、小数点
        if (!/^[\d\s+\-*/().]+$/.test(expr)) throw new Error('公式含非法字符')
        results[key] = Function('"use strict"; return (' + expr + ')')()

  return results
```

### 巴西模板 8 条规则执行示例

输入：premium, beauty, 售价100, 成本25, 重量0.3, 不包邮

```
r1 查表   → 销售佣金费率 = 0.16
r2 百分比 → 销售佣金金额 = 100 × 0.16 = 16.00       (来源=销售佣金费率)
r3 固定值 → 条件不满足(是否包邮≠是)，跳过
r4 查表   → 运费 = 14.35                           (100∈[100,119.99], 0.3∈[0,0.3])
r5 百分比 → 支付手续费 = 100 × 4.8% = 4.80
r6 加总   → 总费用 = 16 + 14.35 + 4.80 = 35.15
r7 公式   → 净利润 = 100 - 35.15 - 25 = 39.85
r8 公式   → 利润率 = 39.85 / 100 = 39.85%
```

### 精确匹配 vs 区间匹配

**精确匹配**（精确）：输入映射中的每个字段的值必须与费率表对应列的值完全相等。用于佣金表（刊登类型=premium AND 类目=beauty → 费率 0.16）。

**区间匹配**（区间）：费率表中含"下限""上限"后缀的列识别为范围列，判断输入值是否在 [下限, 上限] **闭区间**内。用于运费表（售价在 19-48.99 AND 重量在 0-0.3kg → 运费 6.55）。

**区间重叠处理**：运费表的区间边界可能有重叠（如 0-0.3 和 0.3-0.5，重量 0.3 同时命中两行）。引擎规则：**从上到下取第一条匹配行（first match wins）**。建议配置时检查区间不重叠，或在界面给出提醒。

## 设计决策 FAQ

**Q: 为什么是 5 种计算方式（查表/百分比/固定值/加总/公式）？**
A: 跨境电商费用计算覆盖这 5 种。查表（佣金率、运费阶梯）、百分比（固定费率如 4.8% 或动态费率如查表得到的佣金率）、固定值（包邮=0）、加总（总费用=佣金+运费+手续费）、公式（净利润=售价-总费用-成本价）。不需要通用的 set/lookup/branch 动作系统。

**Q: 为什么条件只有单层、无嵌套？**
A: 实际业务中费用规则的条件都很简单（"如果是 premium""如果包邮""如果售价>79"），AND 用同行多条件列，OR 拆多行。不需要 AND/OR 树。

**Q: 输入映射为什么用 `字段键=表列名` 格式而不是 JSON？**
A: Excel 用户可以直接在单元格中编辑逗号分隔的键值对，比 JSON 直观得多。引擎解析也很简单。

**Q: 为什么不保留 presets？**
A: preset 本质是"国家平台+规则集+参数"的捆绑，新设计中模板直接关联国家平台，模板参数提供默认值。去掉了不必要的中间抽象层。

**Q: 复杂条件（如"premium AND (美容 OR 个护)"）如何处理？**
A: 拆成两条规则——规则A：条件=premium+美容，规则B：条件=premium+个护，两条规则费用名称和顺序相同（OR 语义）。如果需要更复杂的嵌套逻辑（极少见），可以增加一个"条件组"列来分组。

# 激活码系统

## 架构概览

```
用户输入激活码 → Tauri 桌面端 → Worker API (/api/activate) → D1 数据库
                    ↑                                      ↑
               本地缓存 7 天宽限期                     Cloudflare 边缘网络
```

- **生成码**：Worker 管理端点或 `pnpm code create`
- **验证激活**：Worker `/api/activate`，查询 D1，绑定设备指纹
- **静默校验**：Tauri 启动时走 `/api/validate`，7 天离线宽限
- **管理码**：`pnpm code` 命令，通过 HTTP 调 Worker 管理端点

## 一、首次部署

### 1. 登录 Cloudflare

```bash
cd worker
pnpm wrangler login          # 浏览器弹窗授权
```

> 如果环境里有 `CLOUDFLARE_API_TOKEN` 会冲突，先 `unset CLOUDFLARE_API_TOKEN`。

### 2. 创建 D1 数据库

```bash
cd worker
pnpm wrangler d1 create profit-db
```

把输出的 `database_id` 填入 `worker/wrangler.toml`：

```toml
[[d1_databases]]
binding = "DB"
database_name = "profit-db"
database_id = "你的database_id"
```

### 3. 初始化表结构（远程）

```bash
cd worker
pnpm wrangler d1 execute profit-db --remote --file=schema.sql
```

### 4. 设置密钥

```bash
cd worker
pnpm wrangler secret put ACTIVATION_SECRET   # 至少 32 位随机字符串，用于签发设备令牌
pnpm wrangler secret put ADMIN_SECRET       # 管理端点认证密钥
```

### 5. 部署 Worker

```bash
cd worker
pnpm wrangler deploy
```

部署后记录 Worker URL（如 `https://profit-api.xu-wei.space`）。

### 6. 配置 Tauri 端 API 地址

编辑 `src-tauri/src/activation.rs` 第 4 行：

```rust
const API_BASE: &str = "https://profit-api.xu-wei.space";
```

编辑 `scripts/code.cjs` 第 15 行：

```js
const API_BASE = 'https://profit-api.xu-wei.space'
```

### 7. 设置本地环境变量

```bash
export PROFIT_ADMIN_SECRET="你设的ADMIN_SECRET"
echo 'export PROFIT_ADMIN_SECRET="你设的ADMIN_SECRET"' >> ~/.zshrc
echo 'export PROFIT_ADMIN_SECRET="1dd87b72-fc38-4058-77f2aa-a4bb9b"' >> ~/.zshrc
```

## 二、日常使用

### 创建激活码

```bash
pnpm code create                           # 1 个，1 台设备，永不过期
pnpm code create 5                         # 5 个
pnpm code create 5 --max-devices=3         # 每码可绑 3 台设备
pnpm code create 1 --expires-in=365d       # 1 年后过期
```

### 查看激活码

```bash
pnpm code all          # 所有激活码（含已用/剩余次数）
pnpm code actived      # 已使用的激活码
pnpm code inactive     # 未使用的激活码
pnpm code check PFT-XXXX-XXXX   # 单个激活码详情 + 绑定设备列表
```

### 删除 / 撤销

```bash
pnpm code delete PFT-XXXX-XXXX
# 无设备绑定 → 物理删除
# 有设备绑定 → 自动转为撤销（revoke），不影响已激活用户
```

## 四、Worker API 端点

### 公开端点

| 端点                 | 用途                            |
| -------------------- | ------------------------------- |
| `POST /api/activate` | 首次激活：验证激活码 + 绑定设备 |
| `POST /api/validate` | 静默校验：已有绑定设备定期验证  |

请求体：`{ "code": "PFT-XXXX-XXXX", "fingerprint": "设备指纹" }`

### 管理端点（受 `ADMIN_SECRET` 保护）

请求需带 `Authorization: Bearer <ADMIN_SECRET>` 头。

| 端点                     | 用途                         |
| ------------------------ | ---------------------------- | ----------- |
| `POST /api/admin/create` | 创建激活码                   |
| `POST /api/admin/delete` | 删除/撤销激活码              |
| `POST /api/admin/list`   | 列出激活码（`?filter=actived | inactive`） |
| `POST /api/admin/check`  | 查看激活码详情 + 设备列表    |

## 五、D1 表结构

### activations（激活码）

| 字段        | 类型    | 说明                       |
| ----------- | ------- | -------------------------- |
| id          | INTEGER | 主键                       |
| code        | TEXT    | 激活码，格式 PFT-XXXX-XXXX |
| status      | TEXT    | active / revoked / expired |
| max_devices | INTEGER | 允许绑定设备数             |
| created_at  | TEXT    | 创建时间                   |
| expires_at  | TEXT    | 过期时间（NULL=永不过期）  |

### devices（设备绑定）

| 字段          | 类型    | 说明                              |
| ------------- | ------- | --------------------------------- |
| id            | INTEGER | 主键                              |
| activation_id | INTEGER | 关联 activations.id               |
| fingerprint   | TEXT    | 设备指纹（hostname+OS 的 SHA256） |
| bound_at      | TEXT    | 绑定时间                          |
| last_seen_at  | TEXT    | 最后活跃时间                      |

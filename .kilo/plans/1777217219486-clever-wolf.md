# 实现计划

## 项目概述

跨平台桌面端利润计算器。Excel 定义字段、选项、规则和查表数据，代码只做通用解释器。面向 Mercado Livre 等平台的跨境卖家使用。

## 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端框架 | Vue 3 (Composition API) | JavaScript，禁止 TypeScript |
| 构建 | Vite + pnpm | |
| 桌面框架 | Tauri v2 | 本地文件读写、系统集成 |
| UI | daisyUI v5 + Tailwind CSS v4 | 已有 `src/style.css` |
| 状态管理 | Pinia | |
| 路由 | vue-router | |
| Excel 解析 | xlsx (SheetJS) | 前端解析，Tauri 负责文件 I/O |
| 工具库 | xe-utils, dayjs | |
| 代码规范 | eslint + @antfu/eslint-config | |

## 阶段 2：数据层 — Excel 读写服务

### 2.1 `配置.xlsx` 工作表结构

| Sheet | 用途 | 关键列 |
|-------|------|--------|
| `overview` | 给人看的总览，代码跳过 | — |
| `presets` | 预设组合 | preset_id, preset_name, country, platform, rule_set_id, enabled |
| `preset_params` | 预设下的参数项 | param_id, preset_id, field_key, param_name, type, unit, option_group_id, default_value, sort, is_required |
| `option_groups` | 选项分组 | group_id, group_name |
| `option_items` | 选项值 | item_id, group_id, label, value, sort, enabled |
| `fields` | 业务字段定义 | field_key, field_name, type, unit, option_group_id, rule_mode |
| `rule_sets` | 规则集入口 | rule_set_id, name, description |
| `rules` | 规则头 | rule_id, rule_set_id, priority, enabled, root_group_id, description |
| `rule_condition_groups` | 条件树分组节点 | group_id, rule_id, parent_group_id, logic, sort |
| `rule_conditions` | 条件叶子 | condition_id, group_id, field_key, operator, value_type, value, value_end |
| `rule_actions` | 命中后动作 | action_id, rule_id, sort, action_type, target_field, config_json |
| `lookup_tables` | 查表目录 | table_id, table_name, match_mode, sheet_name, description |
| `*_table` | 动态查表（如 commission_table） | 列结构由查表自身定义 |

### 2.2 `excel-reader.js` — 核心读取器

负责将 Excel 工作表解析为如下结构化对象：

```js
// 返回结构
{
  presets: [{ preset_id, preset_name, country, platform, rule_set_id, enabled, params: [...] }],
  optionGroups: [{ group_id, group_name, items: [...] }],
  fields: [{ field_key, field_name, type, unit, option_group_id, rule_mode }],
  ruleSets: [{ rule_set_id, name, description }],
  rules: [{ rule_id, rule_set_id, priority, enabled, root_group_id, description, conditions: [...], actions: [...] }],
  lookupTables: [{ table_id, table_name, match_mode, sheet_name, description, rows: [...] }]
}
```

关键点：
- 条件树按 `parent_group_id` 递归组装
- 查表 sheet 按 `lookup_tables.sheet_name` 动态读取，列名来自查表 sheet 第一行
- `config_json` 字段解析为 JSON 对象

### 2.3 `excel-writer.js` — 核心写入器

反向操作：将结构化对象写回 Excel。先在内存中构建完整工作簿，再通过 Tauri 写入文件。

## 阶段 3：规则引擎

### 3.1 `rule-engine.js` — 引擎入口

```js
function execute(fields, rules, lookupTables, userInputs) → { results: {}, errors: [] }
```

执行流程：
1. 按 `priority` 升序遍历 `enabled=true` 的规则
2. 对每条规则，评估条件树是否命中
3. 若命中，按 `sort` 顺序执行 `rule_actions`
4. 返回结果对象和错误列表

### 3.2 条件树评估

递归评估 `rule_condition_groups`：
- `logic = and`：所有子条件 + 子组都满足
- `logic = or`：任一子条件或子组满足
- 叶子条件 (`rule_conditions`)：根据 `operator` 对 `userInputs[field_key]` 与 `value`/`value_end` 比较

支持的操作符：`eq`, `neq`, `gt`, `gte`, `lt`, `lte`, `between`, `in`, `not_in`

### 3.3 动作执行

| action_type | 行为 |
|---|---|
| `set` | `results[target_field] = config_json.value` |
| `lookup` | 从 `config_json.table_id` 对应的查表数据中，按 `config_json.input_map`（如 `{listing_type: userInputs.listing_type, category: userInputs.category}`）匹配行，取 `config_json.output_column` 列的值 |
| `branch` | 评估 `config_json.when` 条件，真则执行 `config_json.then`（subset/loopup 动作），假则执行 `config_json.else` |

查表匹配模式：
- `exact`：等值匹配（所有 input_map 列值相等）
- `range`：区间匹配（input_map 的 range 字段落在 [min, max] 区间）

## 阶段 4：页面实现

### 4.1 预设页 (`PresetPage.vue`)

功能：
- 预设列表（按国家/平台筛选）
- 新建/编辑/删除预设
- 选中预设后，展示参数项可编辑表格
- 参数表格列：参数名称、类型、单位、值/规则、操作
- 支持补齐默认项（从 `fields` 中补全未配置的字段参数）
- 数据读写到 `presets` + `preset_params` 工作表

### 4.2 选项页 (`OptionPage.vue`)

功能：
- 左侧分组列表，右侧选项列表
- 新建/编辑/删除分组
- 选项可编辑表格：显示名称、实际值、排序、启用、备注、操作
- 数据读写到 `option_groups` + `option_items` 工作表

### 4.3 模板页 (`TemplatePage.vue`)

功能：
- 规则集列表
- 选中规则集后展示规则列表
- 编辑规则：规则头 + 条件树 + 动作列表
- 条件树以树形控件展示（AND/OR 节点 + 叶子条件）
- 动作以列表展示，支持添加 set/lookup/branch
- 查表目录管理 + 实际查表 sheet 数据编辑
- 数据读写到全套规则工作表

### 4.4 创建页 (`CreatePage.vue`)

功能：
- 选择预设（下拉/搜索）
- 根据 `preset_params` 动态渲染输入表单
  - `select` 类型：下拉（选项来自 `option_groups/items`）
  - `number` 类型：数字输入 + 单位显示
  - `boolean` 类型：开关
  - `date` 类型：日期选择
- "计算"按钮触发规则引擎
- 结果区域展示 `rule_mode = output` 的字段值
- "保存到列表"按钮 → 写入列表 Excel

### 4.5 列表页 (`ListPage.vue`)

功能：
- 绑定/切换列表 Excel 文件
- 表格展示所有商品记录
- 记录包含：基础信息、计算结果、图片、变体
- 支持删除、编辑记录
- 保存时同步到列表 Excel
- 图片预览

## 阶段 5：Tauri 集成

### 5.1 Rust 后端（`src-tauri/src/main.rs`）

Tauri v2 的 Rust 端保持轻量：

- 注册插件：`tauri-plugin-dialog`, `tauri-plugin-fs`, `tauri-plugin-store`
- 自定义命令（如需要）：
  - `read_excel_file(path)` → 读取文件字节返回到前端
  - `write_excel_file(path, bytes)` → 写入字节到文件

实际上，Tauri v2 的 `@tauri-apps/plugin-fs` 已提供 `readFile`/`writeFile` 基础能力，Excel 解析完全在前端完成。Tauri 只负责：

1. **文件对话框**：打开 `配置.xlsx`、打开/创建 `列表.xlsx`
2. **文件读写**：读字节 → 前端解析；前端序列化 → 写字节
3. **应用设置持久化**：记住上次打开的文件路径

### 5.2 前端 Tauri 调用封装

创建 `src/composables/useFileIO.js`：
- `openConfigExcel()` → 打开配置 Excel 对话框
- `openListExcel()` → 打开列表 Excel 对话框
- `saveConfigExcel(workbook)` → 写入配置 Excel
- `saveListExcel(workbook)` → 写入列表 Excel

### 5.3 应用启动流程

1. 从 `plugin-store` 读取上次配置文件路径
2. 若存在，自动加载 `配置.xlsx`
3. 若不存在，显示空状态引导用户打开文件
4. 加载到 Pinia config store

## 关键约束

1. **禁止 TypeScript** — 所有文件使用 `.js` / `.vue` with `<script setup>`
2. **禁止业务硬编码** — 字段、选项、规则条件、费率一律来自 Excel
3. **引擎不感知业务** — `rule-engine.js` 只认识 field_key、operator、action_type 等抽象概念
4. **配置与记录分离** — `配置.xlsx` 只存配置，`列表.xlsx` 只存商品记录
5. **使用现有风格** — `src/style.css` 已定义 daisyUI 主题，直接沿用

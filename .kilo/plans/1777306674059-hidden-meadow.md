# CountryPlatform 维度表 + Preset 联动重构

## 概述

将 Preset 的 `country` / `platform` 从**自由文本字段**重构为**对 CountryPlatform 维度表的外键引用**，实现集中管理、数据一致性、以及未来多表联动的扩展能力。

## 涉及文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/stores/config.js` | 修改 | 新增 `countryPlatforms` 数组、getter、自动迁移逻辑 |
| `src/utils/excel-reader.js` | 修改 | 新增 `country_platforms` 工作表读取 |
| `src/utils/excel-writer.js` | 修改 | 新增 `country_platforms` 工作表写入，presets 表改用 `cp_id` |
| `src/pages/PresetPage.vue` | 修改 | 国家/平台改为选择已有 CountryPlatform + 管理按钮 |
| `src/pages/CreatePage.vue` | 修改 | 预设选择器、预设信息卡片改为从 CountryPlatform 查表显示 |
| `src/components/common/EditableTable.vue` | 不改 | 复用现有组件 |

## 数据模型

### 新增 `countryPlatforms` 数组

```
countryPlatforms: [{
  cpId:     string,   // 主键，如 "cp_001"
  country:  string,   // 国家名
  platform: string,   // 平台名
  currency: string,   // 货币代码，如 "USD"
  enabled:  boolean,  // 是否启用
}]
```

### Preset 字段变更

| 移除 | 新增 |
|---|---|
| `country` (string) | `cpId` (string, FK → countryPlatforms.cpId) |
| `platform` (string) | |

显示层通过 computed 从 `getCountryPlatform(cpId)` 获取 country/platform 文本。

### Excel 工作表变更

| 工作表 | 变更 |
|---|---|
| `country_platforms` | **新增**，列：`cp_id`, `country`, `platform`, `currency`, `enabled` |
| `presets` | 移除 `country`、`platform` 列，新增 `cp_id` 列 |

### 向后兼容 + 自动迁移

1. Excel 读取时，如果 `presets` 工作表有 `country`/`platform` 列但没有 `cp_id`，则：
   - 扫描所有 preset 的 `(country, platform)` 组合去重
   - 为每个唯一组合自动创建 `CountryPlatform` 条目（`cpId` 自增生成，currency 留空）
   - 设置 `preset.cpId` 指向对应条目
2. Excel 写入时，始终以新格式输出（包含 `cp_id`，不输出 `country`/`platform`），旧格式文件在加载后自动升级

## UI 设计

### PresetPage 工具栏

工具栏新增「国家平台」按钮，点击弹出模态框管理 CountryPlatform 列表：
- EditableTable 显示所有 CountryPlatform，列：`country`、`platform`、`currency`、`enabled`
- 支持增/删/改
- 删除时检查是否有 preset 引用此 `cpId`，有则弹 confirm 警告

### 预设编辑弹窗

- 移除「国家」文本框、「平台」文本框
- 新增「国家平台」下拉选择框，数据源为 `enabled` 的 CountryPlatform 列表
- 显示格式：`美国 (Amazon)` — 即 `country (platform)`
- 下拉旁边加「+」小按钮快速打开新增 CountryPlatform 的模态框

### Preset 列表 & 详情显示

- 列表中 `p.country` → `getCountryPlatform(p.cpId)?.country`
- 详情中 `selectedPreset.country / selectedPreset.platform` → 同上

### CreatePage 适配

- 预设选择器 option 文本中 `(p.country)` → 从 CountryPlatform 查表
- 预设信息卡片中 country/platform → 从 CountryPlatform 查表

## 实现步骤

### 1. 修改 config store

- 新增 `config.countryPlatforms = []`
- 新增 `getCountryPlatform(cpId)` getter
- 新增 `enabledCountryPlatforms` computed
- 新增 `migratePresetCountries()` 方法（自动迁移逻辑）
- 在 `loadFromBuffer` 完成后调用迁移

### 2. 修改 Excel reader

- `src/utils/excel-reader.js` 新增读取 `country_platforms` 工作表
- 解析列：`cp_id` → `cpId`, `country`, `platform`, `currency`, `enabled`(boolean)
- `presets` 工作表同时尝试读取 `cp_id`（新格式）和 `country`/`platform`（旧格式）
- 如果旧格式有 country/platform 但无 cp_id，保留原字段传递给 store 做迁移

### 3. 修改 Excel writer

- 新增 `country_platforms` 工作表输出
- `presets` 工作表用 `cp_id` 替代 `country`/`platform`

### 4. 修改 PresetPage.vue

- 工具栏新增「国家平台」按钮 + 管理模态框（复用 EditableTable）
- 编辑预设弹窗用下拉选择替换国家/平台文本框
- 列表和详情中的 country/platform 改为 computed 查表

### 5. 修改 CreatePage.vue

- preset selector option: `p.country`/`p.platform` → 从 configStore 查表
- preset info card: country/platform → 查表

### 6. 验证

- `pnpm run build` 通过
- 自动迁移：用旧 Excel 测试，验证 CountryPlatform 条目自动创建
- 新建/编辑预设：从下拉选择 CountryPlatform
- 联动：修改 CountryPlatform 的 country 名，所有关联 preset 显示跟随变化

## 未来扩展点

- List 记录可新增 `cpId` 字段，标记记录所属市场
- 定价/物流规则可引用 `cpId` 做市场差异化配置
- `currency` 字段为多币种转换预留

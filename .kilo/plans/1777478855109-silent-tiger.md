# 变体编辑：从 JSON textarea 改为结构化表格

## 根因

当前变体是一个原始 JSON textarea，用户需手写 JSON（如 `{"颜色": ["红","蓝"], "尺寸": ["S","M","L"]}`），容易出错、操作不便。

## 方案

改为表格编辑器，每行一个变体属性，两列：属性名 + 选项值（逗号分隔），自动序列化为 JSON。

## 修改

### 1. `src/stores/create.js`

新增变体表格数据的解析/序列化逻辑：

```js
// 在 variants ref 声明后添加

/** @type {import('vue').Ref<Array<{ id: number, key: string, options: string }>>} */
let variantIdCounter = 0

/** 将 JSON 字符串解析为表格行数组 */
function parseVariantRows(json) {
  if (!json || !json.trim()) return []
  try {
    const obj = JSON.parse(json)
    if (typeof obj !== 'object' || obj === null || Array.isArray(obj)) return []
    return Object.entries(obj).map(([key, val]) => ({
      id: ++variantIdCounter,
      key,
      options: Array.isArray(val) ? val.join(',') : String(val),
    }))
  } catch {
    return []
  }
}

/** 将表格行数组序列化为 JSON 字符串 */
function serializeVariantRows(rows) {
  const obj = {}
  for (const r of rows) {
    if (r.key.trim()) {
      obj[r.key.trim()] = r.options.split(',').map(s => s.trim()).filter(Boolean)
    }
  }
  return Object.keys(obj).length ? JSON.stringify(obj, null, 2) : ''
}

const variantRows = ref(parseVariantRows(variants.value))

// 监听 variants 外部变化（如 selectPreset 重置），同步到 rows
watch(variants, (val) => {
  variantIdCounter = 0
  variantRows.value = parseVariantRows(val)
})

function addVariantRow() {
  variantRows.value.push({ id: ++variantIdCounter, key: '', options: '' })
  variants.value = serializeVariantRows(variantRows.value)
}

function updateVariantRow(id, data) {
  const idx = variantRows.value.findIndex(r => r.id === id)
  if (idx !== -1) {
    variantRows.value[idx] = { ...variantRows.value[idx], ...data }
    variants.value = serializeVariantRows(variantRows.value)
  }
}

function deleteVariantRow(id) {
  variantRows.value = variantRows.value.filter(r => r.id !== id)
  variants.value = serializeVariantRows(variantRows.value)
}

// 在 return 中新增导出
return { ... variantRows, addVariantRow, updateVariantRow, deleteVariantRow }
```

### 2. `src/pages/CreatePage.vue`

将 textarea（第 317-322 行）替换为内联表格：

```html
<table v-if="createStore.variantRows.length" class="table table-sm">
  <thead>
    <tr>
      <th>属性名</th>
      <th>选项值（逗号分隔）</th>
      <th class="w-16">操作</th>
    </tr>
  </thead>
  <tbody>
    <tr v-for="row in createStore.variantRows" :key="row.id">
      <td>
        <input
          :value="row.key"
          class="input input-bordered input-sm w-full"
          placeholder="如：颜色"
          @input="createStore.updateVariantRow(row.id, { key: ($event.target).value })"
        >
      </td>
      <td>
        <input
          :value="row.options"
          class="input input-bordered input-sm w-full"
          placeholder="如：红,蓝,黑"
          @input="createStore.updateVariantRow(row.id, { options: ($event.target).value })"
        >
      </td>
      <td>
        <button class="btn btn-ghost btn-xs text-error" @click="createStore.deleteVariantRow(row.id)">
          删除
        </button>
      </td>
    </tr>
  </tbody>
</table>
<button class="btn btn-ghost btn-sm" @click="createStore.addVariantRow">
  + 添加属性
</button>
```

保留卡片外层结构（`.card`、`.card-body`、标题），仅替换 textarea。

## 数据流

```
表格行 [{ key: "颜色", options: "红,蓝" }, ...]
  ↓ updateVariantRow / addVariantRow / deleteVariantRow
variants (JSON string):  {"颜色": ["红","蓝"], ...}
  ↓ getRecord()
存入 list store / Excel 导出（与原来完全兼容）

回显时：variants JSON → watch → parseVariantRows → variantRows → 表格
```

## 兼容性

- `stores/list.js` 24 行导出头不变
- `ListPage.vue` 178-179 行编辑/查看变体仍显示 JSON 字符串（兼容，后续可优化）
- Excel 导出不变（`variants` 仍是字符串）
- 现有 records 中的 variants JSON 字符串正常解析回表格

# 移除字段页面 + 字段键中文化

## 概述

1. 删除 FieldPage.vue（字段管理页面不再需要 UI 入口，字段通过 Excel 维护）
2. 将 3 个示例 Excel 中所有英文字段键改为中文
3. 同步更新 docs/ 文档中的字段键引用

## 关键理解

- 规则引擎完全数据驱动，源码中**不硬编码**任何字段键（`field_key` 作为字符串在 Excel 的 `fields`、`rule_conditions`、`rule_actions` 等 sheet 间流转）
- 改为中文不会影响代码逻辑，只需确保 Excel 中所有引用的字段键保持一致

## 执行计划

| 操作 | 文件 |
|------|------|
| 删除文件 | `src/pages/FieldPage.vue` |
| 移除路由 | `src/router/index.js` → 删除 `field` 路由 |
| 移除侧边栏入口 | `src/layouts/DefaultLayout.vue` → 删除 `navItems` 中的字段项 |

### 2. Excel 字段键中文化（3 份 xlsx 副本）

需要同时更新 `docs/excel-rule-engine-example.xlsx`、`docs/excel-rule-engine-example copy.xlsx`、`public/examples/excel-rule-engine-example.xlsx`。

#### 2.1 字段键映射表

| 当前（英文） | 改为（中文） | 类型 |
|-------------|-------------|------|
| `listing_type` | `刊登类型` | input 字段 |
| `category` | `类目` | input 字段 |
| `sale_price` | `售价` | input 字段 |
| `shipping_weight` | `重量` | input 字段 |
| `is_free_shipping` | `是否包邮` | input 字段 |
| `commission_rate` | `销售费率` | output 字段 |
| `shipping_fee` | `运费` | output 字段 |
| `seller_shipping_cost` | `卖家运费` | output 字段 |
| `fixed_fee` | `固定费用` | output 字段 |
| `fixed_additional_fee` | `固定附加费` | output 字段 |
| `output_value` | `输出值` | 查表输出列 |
| `range_1` / `range_1_min` / `range_1_max` | `范围1` / `范围1下限` / `范围1上限` | 查表列 |
| `range_2` / `range_2_min` / `range_2_max` | `范围2` / `范围2下限` / `范围2上限` | 查表列 |

#### 2.2 各 Sheet 需修改内容

**`fields` sheet** — `field_key` 列全部替换为中文

**`rule_conditions` sheet** — `field_key` 列替换：`listing_type`→`刊登类型`，`sale_price`→`售价`，`category`→`类目`

**`rule_actions` sheet** — 两处修改：
- `target_field` 列：`commission_rate`→`销售费率`，`shipping_fee`→`运费`，`seller_shipping_cost`→`卖家运费`
- `config_json` JSON 字符串内替换：
  - `input_map` 的 key/value 中的字段键
  - `output_column` 的值
  - `when.field` 的值（`is_free_shipping`→`是否包邮`）

**`commission_table` sheet** — 列头：`listing_type`→`刊登类型`，`category`→`类目`，`output_value`→`输出值`

**`shipping_cost_table` sheet** — 列头：`range_1_min`→`范围1下限`，`range_1_max`→`范围1上限`，`range_2_min`→`范围2下限`，`range_2_max`→`范围2上限`，`output_value`→`输出值`

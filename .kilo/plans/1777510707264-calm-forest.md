# SKU + 变体 + 图片 + 计算联动重构

## 业界参考

主流电商平台（Shopify、淘宝、Amazon）的产品变体管理模式：

| 层级 | 内容 |
|------|------|
| 产品级 | 名称、描述、分类、**共享图片**、**默认成本/重量** |
| 变体属性 | 颜色: [红,蓝]、尺寸: [S,M] → 自动生成所有组合 |
| SKU级 | 每组合独立：**SKU款号**、**成本**、**重量**、**价格/利润**、**专属图片** |

不同 SKU 的成本、重量可能不同（如 XL 码成本高、重量大），因此计算结果（售价、利润）也需逐 SKU 计算。

## 当前问题

```
基础信息: [name, sku, cost, weight]  ← 产品级，cost/weight 不在 rule-engine 中运算
                                                  ↓
输入参数: [preset fields...]           ← rule-engine 的输入源
                                                  ↓
rule-engine(execute) → results        ← 产品级，一次计算
图片:    [ImageUploader]              ← 产品级
变体:    [attribute rows]             ← 仅定义属性维度，不生成 SKU
```

**核心割裂**：
1. `cost`/`weight` 在 `basicInfo`（产品级），不进 rule-engine，无法参与运算
2. SKU款号、图片与变体无关联
3. 无法逐 SKU 计算不同的售价和利润

## 目标设计

### 数据流（重构后）

```
基础信息: [name, base_sku, default_cost, default_weight]
变体属性: [颜色: 红,蓝] [尺寸: S,M]
         ↓ auto-generate
SKU表:    ┌───────┬─────────┬──────┬──────┬──────┐
         │ 组合  │ SKU款号 │ 成本 │ 重量 │ 图片 │
         │ 红,S  │ TS-R-S  │  12  │  150 │ [+]  │
         │ 蓝,M  │ TS-B-M  │  14  │  180 │ [+]  │
         └───────┴─────────┴──────┴──────┴──────┘
              ↓ per-SKU
rule-engine(execute) × N → skuResults[]
              ↓ per-SKU
结果表:   ┌───────┬──────┬────────┐
         │ 组合  │ 售价  │ 利润   │
         │ 红,S  │  50  │  10    │
         │ 蓝,M  │  55  │  12    │
         └───────┴──────┴────────┘
```

### 数据结构

新增 `skuData` 字段（JSON，存 Excel）：

```json
{
  "红,S": {
    "sku": "TS-R-S",
    "images": "/path/red_s.jpg,https://...",
    "overrides": {
      "cost": "12",
      "weight": "150",
      "shippingFee": "5.00"
    }
  },
  "红,M": {
    "sku": "TS-R-M",
    "images": "",
    "overrides": {}
  },
  "蓝,S": {
    "sku": "TS-B-S",
    "images": "/path/blue_s.jpg",
    "overrides": {
      "cost": "14",
      "重量": "150"
    }
  }
}
```

- key：变体组合（属性值按定义顺序用逗号连接）
- `sku`：为空时自动生成 `{baseSku}-{值1}-{值2}`
- `images`：每 SKU 独立的图片（逗号分隔，同原格式）
- `overrides`：**任意预设字段的每 SKU 覆盖值**（cost、weight、shippingFee、packagingFee 等——不限于固定字段，完全由预设参数决定）。为空 `{}` 表示全部继承产品级值

**合并规则**（计算时）：
```
productDefaults = { ...basicInfo(cost/weight), ...userInputs(preset params) }
skuInputs = { ...productDefaults, ...skuData[comboKey].overrides }
→ execute(fields, rules, lookupTables, skuInputs)
```
任何字段只要在 `overrides` 中有值就覆盖，否则沿用产品级默认。

### 每 SKU 计算逻辑

```
productDefaults = { ...basicInfo, ...userInputs }  // 产品级全量默认值

for each SKU combo:
  mergedInputs = { ...productDefaults, ...skuData[comboKey].overrides }
  { results } = execute(fields, rules, lookupTables, mergedInputs)
  skuResults[comboKey] = results
```

- `execute()` 本身**无需修改**（纯函数）
- 仅需循环调用，每次传入合并后的 inputs（产品级默认 + SKU 覆盖）
- `overrides` 中的字段名必须与预设参数字段名一致（或 basicInfo 字段名）
- 计算结果**实时显示**在 SKU 表格的结果列中
- skuResults 不单独持久化到 Excel（可重算），仅在内存中保留

### 产品级结果

- **无变体时**：保持原有单次计算行为，results 存产品级
- **有变体时**：产品级 results 面板隐藏（或显示汇总），各 SKU 结果直接在 SKU 表中展示

### UI 布局（CreatePage）

```
┌─ 基础信息 ──────────────────────────────────────┐
│ name | base_sku | default_cost | default_weight │
└─────────────────────────────────────────────────┘
┌─ 输入参数 ──────────────────────────────────────┐
│ [preset fields...]                               │
│  ↳ 这些值作为所有 SKU 的默认输入                  │
└─────────────────────────────────────────────────┘
┌─ 变体 ───────────────────────────────────────────┐
│ [+ 添加属性]                                     │
│ 颜色: [红,蓝,黑]  尺寸: [S,M,L]                 │
│                                                   │
│ 显示字段: [成本] [重量] [运费] [+ 选择字段]      │  ← 可配置列
│                                                   │
│ ┌─ SKU 明细表（自动生成）───────────────────────┐ │
│ │ 组合  │ SKU款号 │图片│ 成本 │ 重量 │ 售价│利润│ │
│ │ 红,S  │ TS-R-S  │ [+]│  12  │  150 │  50│ 10 │ │
│ │ 红,M  │ TS-R-M  │ [+]│  12  │  180 │  52│ 11 │ │
│ │ 蓝,S  │ TS-B-S  │ [+]│  14  │  150 │  55│ 12 │ │
│ │ 蓝,M  │ TS-B-M  │ [+]│  14  │  180 │  57│ 13 │ │
│ └───────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
┌─ 图片（共享）─ 无变体时独立显示 ───────────────┐
│ [ImageUploader]                                  │
└─────────────────────────────────────────────────┘
```

**表格列逻辑**：
- 固定列：变体组合（只读）、SKU款号（可编辑）、图片（每SKU独立ImageUploader）
- 可配置列：一个下拉多选组件，列出所有预设参数字段（cost + weight + 所有 userInput 字段），用户勾选哪些字段就显示哪些列。默认勾选 `cost`、`weight`
- 结果列（只读）：自动追加在每个 SKU 行右侧，显示该 SKU 的计算结果
- 空白 cell = 继承产品级默认值，显示为灰色 placeholder
- 用户填入值 = 覆盖，正常文字颜色

### 行为规则

| 场景 | 行为 |
|------|------|
| 无变体属性 | 保持原来行为：产品级 images + 单次计算 → results |
| 新增/删除/修改变体属性 | 即时重新生成 SKU 组合，保留已有 SKU 数据（matches by comboKey），清理过期组合 |
| SKU 款号为空 | 自动填充 `{baseSku}-{值1}-{值2}` |
| SKU override 字段为空 | 计算时继承产品级默认值（来自 basicInfo 或 userInputs） |
| 用户勾选/取消勾选显示字段 | 表格列即时增减，不影响数据 |
| 计算按钮 / 自动计算 | 循环所有 SKU，合并产品级默认 + SKU override → 调用 execute() → 更新 skuResults |
| 删除所有变体属性 | 回退到产品级模式，隐藏 SKU 表，恢复原 results 面板 |
| 保存到列表 | `getRecord()` 包含 `skuData` JSON（仅 persist sku+images+overrides，results 不存） |

## 实施步骤

### Step 1: 数据层 — `src/stores/create.js`

**新增状态**：
- `skuData` ref（初始 `'{}'`，持久化到 record）
- `skuResults` ref（每 SKU 的计算结果，仅内存，不持久化）

**新增 computed**：
- `generatedSkuCombos`：从 variantRows 笛卡尔积生成 `[{key: "红,S", values: {颜色:"红",尺寸:"S"}}, ...]`
- `allOverrideFieldKeys`：合并 basicInfo keys（cost/weight）+ 所有 preset param fieldKey，用于 SKU 表字段选择器

**新增/修改方法**：
- `refreshSkuData()`：当 variantRows 变化时调用，合并已有 skuData（按 comboKey）与新组合
- `updateSkuField(comboKey, field, value)`：更新某 SKU 的 `sku` / `images` 或 `overrides[field]`
- `calculate()`：重构 — 产品级计算 + 每 SKU 循环计算（合并 productDefaults + sku.overrides）
- `getRecord()`：增加 `skuData: skuData.value`（results 不存入 skuData）
- `reset()` / `selectPreset()`：重置 skuData 和 skuResults

### Step 2: SKU 表格组件 — 新建 `src/components/common/SkuTable.vue`

**Props**:
- `skuCombos: array`（生成的组合列表）
- `skuData: object`（解析后的 JSON，含 sku/images/overrides）
- `skuResults: object`（每 SKU 计算结果）
- `baseSku: string`
- `defaultValues: object`（产品级默认值 `{cost, weight, ...userInputs}`，用于 placeholder）
- `allFieldKeys: string[]`（所有可覆盖的字段键）
- `selectedFieldKeys: string[]`（用户选择显示的字段列）

**Emits**:
- `update:selectedFieldKeys` — 列选择变更
- `update-sku-field(comboKey, field, value)` — 单字段更新
- `update-sku-images(comboKey, imagesString)` — 图片更新

**模板**：
- 顶部：字段选择器（DaisyUI dropdown + 多选 checkbox）
- 固定列：变体组合（只读）| SKU款号（input）| 图片（ImageUploader）
- 可选列：由 `selectedFieldKeys` 决定，每个是一个 input，placeholder 显示默认值
- 结果列：遍历 skuResults keys，每 SKU 显示对应结果（只读）
- 空 override 的 input 为灰色 placeholder 显示"继承：xxx"

### Step 3: UI 重构 — `src/pages/CreatePage.vue`

- 变体区域：在属性编辑表下方插入 `<SkuTable>`（`v-if` 有有效变体时）
- 计算结果面板：有变体时隐藏（改为 SKU 表展示），无变体时保持原样
- "图片"卡片：有变体时标签改为"图片（共享）"，无变体时保持原样
- 计算按钮触发后，同时更新 product-level results 和 skuResults

### Step 4: ListPage — `src/pages/ListPage.vue`

- 列表表格：新增 `skuData` 列（默认不显示，可由用户通过表头配置选择）
- 详情查看：解析 `skuData` + `variants`，展示 SKU 明细表（款号 + 图片 + 价格等）
- 编辑模式：
  - `variants` 字段：改用结构化编辑（复用变体属性行）
  - `skuData` 字段：显示 SKU 表编辑（复用 SkuTable）

### Step 5: 存储层 — `src/stores/list.js`

- `defaultHeaders` 增加 `{ key: 'skuData', prop: 'skuData' }`

### Step 6: 规则引擎适配 — `src/services/rule-engine.js`

- **无需修改**：`execute()` 是纯函数，已在 Step 1 中通过循环调用实现每SKU计算

### Step 7: 向后兼容

- 旧记录无 `skuData` → `JSON.parse('')` 异常捕获 → 视为 `{}`，等同于无变体模式
- 旧的 `sku` 字段语义：无变体时保持原义（产品款号），有变体时作为 baseSku 前缀
- 旧的 `images` 字段（产品级共享图）：有变体时保留作为"共享图片"，无变体时同原行为

- `prepare(text, font, options?)` → `PreparedText`
- `layout(prepared, maxWidth, lineHeight)` → `{ height, lineCount }`

#### Step 2: 创建 `src/utils/textMeasure.js`

**修正规划文档的问题**:
- 默认字体改为命名字体而非 `system-ui`（Pretext 明确警告 macOS 上 system-ui 不准确）
- 项目 CSS (`style.css`) 未显式设置 font-family，依赖 Tailwind/DaisyUI 默认值
- Tailwind v4 默认 sans 字体栈: `ui-sans-serif, system-ui, sans-serif, "Apple Color Emoji"...`
- **但 Pretext 不支持 `system-ui`**，在 macOS 上测量偏差
- `html { font-size: 20px }` — 项目基础字号是 20px，不是 14px！
- 解决方案: Pretext 使用 `-apple-system, BlinkMacSystemFont, "Helvetica Neue"` 等 macOS 命名字体，Windows 用 `"Segoe UI"`

```js
import { prepare, layout, prepareWithSegments, layoutWithLines } from '@chenglou/pretext'

const DEFAULT_FONT = '20px -apple-system, BlinkMacSystemFont, "Helvetica Neue", "Segoe UI", Roboto, Arial, sans-serif'

export function measureTextHeight(text, maxWidth, lineHeight = 28, font = DEFAULT_FONT) {
  if (!text) return 0
  const prepared = prepare(String(text), font)
  const { height } = layout(prepared, Number(maxWidth) || 200, lineHeight)
  return height
}

export function measureTextLines(text, maxWidth, lineHeight = 28, font = DEFAULT_FONT) {
  if (!text) return { lines: [], height: 0, lineCount: 0 }
  const prepared = prepareWithSegments(String(text), font)
  return layoutWithLines(prepared, Number(maxWidth) || 200, lineHeight)
}

export function getEllipsisText(text, maxWidth, font = DEFAULT_FONT, maxChars = 20) {
  if (!text || text.length <= maxChars) return { text, truncated: false }
  const fullHeight = measureTextHeight(text, maxWidth)
  const shortText = text.slice(0, maxChars) + '...'
  const shortHeight = measureTextHeight(shortText, maxWidth)
  if (fullHeight > shortHeight) {
    return { text: shortText, truncated: true }
  }
  return { text, truncated: false }
}
```

**补充**: 需添加缓存机制（Pretext `prepare()` 是一次性工作，不应重复调用同文本）
- 使用 `Map` 缓存 `PreparedText` 对象，key 为 `${text}|${font}`
- 提供 `clearTextMeasureCache()` 方法，在大量数据切换时清理

### 第二阶段：核心场景

#### Step 4: SKU 表格动态行高（原场景 2）

**位置**: `ListPage.vue` SKU 表格 `<tr>` (约 line 362)

当前 SKU 表格使用固定行高（DaisyUI `table table-sm`），部分单元格内容较长。

**实施方案**:
- SKU 行中的文本单元格（如备注、描述）使用 `measureTextHeight` 计算所需高度
- 整行高度取所有单元格最大高度: `Math.max(32, maxCellHeight + 8)`
- 用 `:style="{ height: rowHeight + 'px' }"` 动态设置

**注意**: SKU 数量通常较少（几十条），动态行高不会带来性能问题

### 第三阶段：高级功能

#### Step 6: 列表虚拟滚动（原场景 1）

**位置**: `ListPage.vue` 商品记录 `<tr v-for="(row, idx) in listStore.records">` (约 line 487)

这是最有价值的场景。当记录数 > 500 时，一次性渲染所有行会导致卡顿。

**实施方案 - 手写虚拟滚动（推荐）**:

不需要引入额外库。利用 Pretext 预计算每行高度，实现精准虚拟滚动：

1. 使用 `measureTextHeight` 或固定行高估算每行高度
2. 计算可见区域范围: `scrollTop / itemHeight` → `startIndex`, `containerHeight / itemHeight` → `count`
3. 只渲染可见范围的行（加 buffer 2-3 行）
4. 用占位 `<div>` 设置总高度维持滚动条

**关键组件结构**:
```vue
<div class="overflow-auto" :style="{ maxHeight: '600px' }" @scroll="onScroll">
  <div :style="{ height: totalHeight + 'px', position: 'relative' }">
    <table :style="{ transform: `translateY(${offsetY}px)` }">
      <tr v-for="row in visibleRecords">...</tr>
    </table>
  </div>
</div>
```

**与拖拽排序兼容**:
- 当前记录表格使用 `useSortable` (@shopify/draggable) 实现拖拽排序
- 虚拟滚动只渲染可见行，拖拽仅对可见行生效
- 不可见行无法拖拽，需补充"移到顶部/底部"按钮
- 阈值策略: ≤ 100 条 → 全量渲染+拖拽；> 100 条 → 虚拟滚动+可见行拖拽+移动按钮

**动态行高方案**:
- 商品记录行高可能因列内容不同而变化
- 预计算每行所有列文本的最大高度
- 缓存行高数组，避免重复计算
- 总高度 = 所有行高之和

#### Step 7: 低优先级场景（原场景 4、5）

- **编辑列弹窗列名截断**: `ColEditorModal.vue` 中的 truncate，CSS 方案已足够
- **表单标签动态宽度**: `FieldInput.vue` 的 DaisyUI label 布局，当前已满足需求

**建议**: 这两个场景暂不实施，优先级确实低。

## 需用户确认的问题

1. **虚拟滚动与拖拽排序**: 可以共存。虚拟滚动只渲染可见行，拖拽排序仅对可见行生效。不可见行不在 DOM 中无法拖拽，需补充"移到顶部/底部"按钮实现跨范围移动。记录数 ≤ 100 时全量渲染+拖拽，> 100 时虚拟滚动+可见行拖拽+移动按钮。

2. **CSS 字体配置**: 已确认 — 项目 CSS 未显式设 font-family，依赖 Tailwind/DaisyUI 默认栈。`html { font-size: 20px }`，基础字号 20px。Pretext 字体需使用命名字体避免 `system-ui`，默认 lineHeight 应为 28px (20px 字号 × 1.4)。表格中 `table-xs` (12px) / `table-sm` (14px) 等尺寸需分别指定。

3. **长文本智能省略**: 用户确认暂不实施，CSS truncate 已满足需求。

4. **虚拟滚动行高策略**: 商品记录是否需要动态行高（每行高度不同），还是可以使用固定行高？固定行高实现更简单，动态行高需预计算每行。

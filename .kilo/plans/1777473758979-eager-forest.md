# 诊断：输入值跨字段串扰

## 根因假设

当前 `FieldInput` 组件通过 `fieldKey` 绑定到 `createStore.userInputs[fieldKey]`。如果同一 preset 下有多个 param 的 `fieldKey` 相同，这些输入框就会共享同一个 store 值，表现即为"输入 A 框，B 框跟着变"。

数据来源：Excel 配置中 `preset_params` 表的 `field_key` 列。

## 诊断步骤

### 1. 在 `paramFields` computed 中检测重复 fieldKey

在 `src/pages/CreatePage.vue` 的 `paramFields` computed 中添加检测逻辑：

```js
const paramFields = computed(() => {
  const fields = presetParams.value.map((param) => {
    const field = configStore.getField(param.fieldKey)
    return {
      ...param,
      fieldName: field?.fieldName || param.paramName || param.fieldKey,
      type: param.type || field?.type || 'text',
      unit: param.unit || field?.unit || '',
      optionGroupId: param.optionGroupId || field?.optionGroupId || '',
      required: param.isRequired,
    }
  })

  // 检测重复 fieldKey
  const keys = fields.map(f => f.fieldKey)
  const dupes = keys.filter((k, i) => keys.indexOf(k) !== i)
  if (dupes.length > 0) {
    console.warn('[CreatePage] 检测到重复 fieldKey:', [...new Set(dupes)], '在预设:', createStore.selectedPresetId)
  }

  return fields
})
```

### 2. 用户验证

1. 打开浏览器控制台（F12 → Console）
2. 选择预设（选择出现问题的那个预设）
3. 观察控制台输出
   - 如果看到黄色警告 `[CreatePage] 检测到重复 fieldKey: [...]` → 确认是 Excel 配置中 `field_key` 重复
   - 如果没有警告但问题仍存在 → 需要进一步排查

### 3. 后续修复方向（待确认根因后执行）

#### Case A：确认是重复 fieldKey

- 用户在 Excel 的 `preset_params` 表中修正重复的 `field_key`
- 或增加代码层去重：`paramFields` 中过滤掉 fieldKey 重复的 param（保留第一个）

#### Case B：没有重复 fieldKey（罕见）

- 增加更详细的日志：在 `updateInput` 中 log 每次 `fieldKey` 和 `value`
- 检查是否 Vue 响应式追踪问题，可能需要改用 `shallowRef` + 手动 trigger

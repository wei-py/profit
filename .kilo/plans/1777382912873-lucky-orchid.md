# OptionPage: 修复新增/编辑选项项的核心缺陷

## Root Cause

`itemValue` 同时承担两个角色：**用户可编辑的数据字段** + **EditableTable 行唯一标识符（idKey）**。这导致两个 Bug：

### Bug 1: 新增行互相影响
`handleItemAdd` 创建新项时 `itemValue: ''`。如果连续添加多行，它们的 `itemValue` 都是空字符串。`EditableTable` 用 `itemValue` 作为 `idKey`，`isEditing()` 判断 `editingRowId === row['itemValue']` 对多行同时为 `true`，导致多个行共享同一个 `editData`，编辑任意一行的输入其他行都会同步变化。

### Bug 2: 编辑 itemValue 后保存无效
修改 `itemValue` 后点击保存，`saveEdit()` 发出 `editData`（包含新的 `itemValue`），`handleItemUpdate(row)` 用 `findIndex(i => i.itemValue === row.itemValue && i.groupId === row.groupId)` 查找原始项。但原始项的 `itemValue` 还是旧值，`findIndex` 返回 `-1`，查找失败，更新被静默忽略。

## Solution

引入 `_uid` 作为稳定的行标识符，与 `itemValue` 解耦：

1. **OptionPage.vue 初始化**：config 加载后，给所有已有项追加 `_uid`（不持久化到 Excel，仅运行时使用）
2. **handleItemAdd**：新项生成唯一 `_uid`，`itemValue` 恢复为空字符串
3. **id-key** 改为 `'_uid'`
4. **handleItemUpdate / handleItemDelete** 改用 `_uid` 查找

## Implementation

### 1. 添加 `watch(loaded)` + `ensureUids`

```js
function ensureUids() {
  for (const group of configStore.config.optionGroups) {
    for (const item of (group.items || [])) {
      if (!item._uid) {
        item._uid = `uid_${Date.now()}_${Math.random().toString(36).slice(2, 8)}`
      }
    }
  }
}

watch(() => configStore.loaded, (loaded) => {
  if (!loaded) return
  ensureUids()
  const cached = localStorage.getItem(CACHE_GROUP_KEY)
  if (cached && configStore.config.optionGroups.some(g => g.groupId === cached)) {
    selectedGroupId.value = cached
  }
}, { immediate: true })
```

`onMounted` 可以删除，因为 `watch` + `immediate: true` 已覆盖加载完成 + 已加载两种场景。

### 2. handleItemAdd — 回退 itemValue 为空，添加 _uid

```js
function handleItemAdd() {
  if (!selectedGroup.value) return
  const newItem = {
    _uid: `uid_${Date.now()}_${Math.random().toString(36).slice(2, 8)}`,
    groupId: selectedGroupId.value,
    itemValue: '',
    itemLabel: '',
    sort: (selectedGroup.value.items || []).length + 1,
    enabled: true,
    remark: '',
  }
  selectedGroup.value.items.push(newItem)
}
```

### 3. 修改 id-key

```html
<EditableTable
  id-key="_uid"
  ...
/>
```

### 4. handleItemUpdate — 改用 _uid 查找

```js
function handleItemUpdate(row) {
  if (!selectedGroup.value) return
  const idx = selectedGroup.value.items.findIndex(i => i._uid === row._uid)
  if (idx !== -1) {
    selectedGroup.value.items[idx] = row
  }
}
```

### 5. handleItemDelete — 改用 _uid 查找

```js
function handleItemDelete(row) {
  if (!selectedGroup.value) return
  selectedGroup.value.items = selectedGroup.value.items.filter(i => i._uid !== row._uid)
}
```

## Verification

- `pnpm run build` 通过
- 连续点击"新增一行"两次，两个空行独立编辑互不影响
- 编辑某行的 `itemValue` 并保存，值正确更新
- 编辑某行的 `itemLabel`（显示名）并保存，值正确更新
- Excel 导出/导入后，`_uid` 不会出现在 Excel 中（导出仅含 `itemValue`/`itemLabel`/`sort`/`enabled`/`remark`/`groupId`）

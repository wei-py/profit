# CountryPlatform + Preset 重构：代码修复 & 示例 Excel 跟进

## 概述

前一轮已实现 CountryPlatform 维度表 + Preset cpId 外键重构的核心代码，但存在两个遗留问题：

1. **致命 bug**：`excel-reader.js` 的 `readConfigWorkbook` 中 result 初始化缺少 `countryPlatforms: []`，导致加载不含 `country_platforms` 工作表的 Excel 时，`config.value.countryPlatforms` 被覆盖为 `undefined`，触发 `enabledCountryPlatforms` computed 崩溃
2. **示例 Excel 未跟进**：`excel-rule-engine-example.xlsx`（3 份副本）缺少 `country_platforms` 和 `presets` 工作表，与代码实现脱节

## 关键理解

### 数据关系链

```
CountryPlatform (维度表)     Preset (配置入口)        RuleSet (规则集)
┌─────────────────────┐    ┌────────────────────┐    ┌─────────────────┐
│ cpId    ← FK ←──────┼────│ cpId               │    │                 │
│ country              │    │ presetName         │    │ ruleSetId ←──── │
│ platform             │    │ ruleSetId ─────────┼───→│ name            │
│ currency             │    │ enabled            │    │ rules[]         │
│ enabled              │    │ params[]           │    │                 │
└─────────────────────┘    └────────────────────┘    └─────────────────┘
```

- CountryPlatform = 市场定义（国家 + 平台 + 货币），如「巴西 / Mercado Livre / BRL」
- Preset = 把市场绑定到规则集，形成「某市场下的某套定价/物流规则」
- 当前示例 Excel 只有 `rule_sets`（`ml_br_general`），缺少 CountryPlatform 维度表和 Preset 入口层

## 执行计划

### 1. 修复 excel-reader.js 致命 bug

`src/services/excel-reader.js:105-112` — 在 `readConfigWorkbook` 的 `result` 初始化对象中添加 `countryPlatforms: []`。

```js
// 修复前
const result = {
  presets: [],
  optionGroups: [],
  fields: [],
  ruleSets: [],
  rules: [],
  lookupTables: [],
}

// 修复后
const result = {
  presets: [],
  optionGroups: [],
  fields: [],
  ruleSets: [],
  rules: [],
  lookupTables: [],
  countryPlatforms: [],
}
```

**为什么这是必要的**：`loadFromBuffer` 中执行 `config.value = data`，整体替换 ref 值。如果 Excel 不含 `country_platforms` sheet，`data.countryPlatforms` 为 `undefined`，覆盖掉 `ref` 初始值中的 `[]`，导致 `enabledCountryPlatforms` computed 中 `.filter` 调用报错。

### 2. 更新示例 Excel（3 份副本）

需要同步更新的文件：
- `docs/excel-rule-engine-example.xlsx`
- `docs/excel-rule-engine-example copy.xlsx`
- `public/examples/excel-rule-engine-example.xlsx`

#### 2.1 新增 `country_platforms` 工作表

| cp_id | country | platform | currency | enabled |
|-------|---------|----------|----------|---------|
| cp_001 | 巴西 | Mercado Livre | BRL | true |
| cp_002 | 墨西哥 | Mercado Livre | MXN | true |

说明：
- `cp_id` 与代码中的 `cpId` 字段对应（驼峰转换由 reader 处理）
- 巴西用 BRL（雷亚尔），墨西哥用 MXN（比索）
- 两个市场都指向同一个 `rule_set_id = ml_br_general`（示例中只有一个规则集，代表通用定价逻辑可复用）
- 不要用中文列名，保持与其他 sheet 一致的 snake_case 英文列名

#### 2.2 新增 `presets` 工作表

| preset_id | preset_name | cp_id | rule_set_id | enabled |
|-----------|-------------|-------|-------------|---------|
| preset_001 | 巴西ML通用 | cp_001 | ml_br_general | true |
| preset_002 | 墨西哥ML通用 | cp_002 | ml_br_general | true |

说明：
- `preset_name` 用中文命名，面向使用者
- `cp_id` 引用 `country_platforms.cp_id`
- `rule_set_id` 引用唯一的 `ml_br_general` 规则集
- **不包含** `country` 和 `platform` 列（新格式只含 `cp_id`）

#### 2.3 暂不新增 `preset_params` 工作表

当前示例的字段定义足够通用（listing_type、category、sale_price 等），无需参数覆盖。`preset_params` 留作后续扩展用。

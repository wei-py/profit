# Driver.js 用户引导集成 v2

## 概述

集成 [Driver.js](https://driverjs.com) + daisyUI tooltip 为"利润工具"添加两层帮助系统：
1. **引导教程** — 侧边栏"帮助"按钮 → 弹出菜单选择"当前页面引导"或"应用概览"，Driver.js 分步引导
2. **悬停提示** — 页面右下角固定 `?` 浮动按钮，hover 弹出当前页面功能概述 tooltip

## 实现步骤

### 1. 重写 `src/composables/useTour.js`

- 导入 `driver` / `driver.css`（不变）
- **移除** `commonSteps` 自动追加到每页引导的逻辑
- 新增独立 `overviewSteps`（仅侧边栏/折叠/主题3步）
- 各页面步骤大幅细化（见下方步骤表）
- 导出 `startTour(type)` — type 为 `'overview'` 或路由名 `'preset'|'option'|...`
- 保留 `stopTour()`

### 2. 修改 `DefaultLayout.vue` — 帮助按钮改为弹出菜单

"帮助"按钮点击后不再直接启动引导，而是弹出 daisyUI dropdown 菜单：
- 选项1：**「当前页面引导」** → `startTour(routeName)`
- 选项2：**「应用概览」** → `startTour('overview')`
- 使用 `<div class="dropdown">` 实现，点击展开/收起

### 3. 新增浮动 `?` 按钮（DefaultLayout.vue）

- 固定在页面右下角 `position: fixed; bottom: 1rem; right: 1rem`
- 圆形按钮，显示 `?` 图标
- 使用 daisyUI `.tooltip` 类，hover 时弹出当前页面的功能概述文字
- 内容按路由名映射，例如：
  - `/preset` → "预设管理：创建和管理利润计算预设，定义参数列表"
  - `/option` → "选项分组：为选择型字段提供下拉选项，如颜色、尺寸"
  - `/field` → "字段定义：定义所有可用字段的键、名称、类型"
  - `/template` → "规则模板：配置条件树与动作，管理查找表"
  - `/create` → "新建记录：选择预设填写参数，一键计算利润"
  - `/list` → "记录列表：查看、编辑和删除已保存的记录"
  - `/` (fallback) → "利润工具：管理产品利润计算的配置与记录"

#### 公共（overview）— 仅3步

### 5. 浮动 `?` 按钮实现

- 在 `DefaultLayout.vue` 的 `<template>` 中添加固定定位元素
- 使用 daisyUI tooltip：`<div class="tooltip tooltip-left" :data-tip="pageTooltip">` 
- 用 `computed` 按 `route.name` 映射 tooltip 文本
- 样式：`position: fixed; bottom: 1rem; right: 1rem; z-index: 50;` 圆形按钮

### 6. Driver.js 样式微调（可选）

在 `src/style.css` 中增加少量覆盖，使 driver.js popover 与 daisyUI 主题一致：
```css
.driver-popover {
  font-family: inherit;
}
.driver-popover button {
  font-family: inherit;
}
```
保持 driver.js 默认样式为主，不做过重定制。

## 验证方式

1. `npm run build` 确保构建通过
2. 点击侧边栏"帮助"验证下拉菜单出现
3. 选"应用概览"验证3步引导（导航/折叠/主题）
4. 逐页面选"当前页面引导"验证细化步骤正常
5. hover 右下角 `?` 按钮验证 tooltip 文案正确
6. `.driver-popover` 文字大小受 `html { font-size: 17px }` 控制，保持一致

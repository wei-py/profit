# Fix: DaisyUI menu button hover 显示不全（根本原因）

## 根因

分析 `node_modules/daisyui/daisyui.css` 发现 DaisyUI v5 的 `.menu` 设置了 `width: fit-content`，导致菜单仅与内容同宽。`<button>` 的 hover 背景也因此无法撑满侧边栏完整宽度。

之前的修复（去掉 `flex items-center`、改用 `menu-active`）是正确的，但没有解决宽度问题。

## 修改文件

### `src/layouts/DefaultLayout.vue`

第 31 行的 `<ul>` 需要修改：

**Before:**
```html
<ul class="menu menu-vertical px-2 gap-1">
```

**After:**
```html
<ul class="menu menu-vertical w-full">
```

变更：
1. 去掉 `px-2` —— DaisyUI 的 `.menu` 已有 `padding: 0.5rem`，`px-2` 叠加了多余内边距
2. 去掉 `gap-1` —— DaisyUI 的 `.menu` 已内置项目间距
3. 添加 `w-full` —— 覆盖 DaisyUI 的 `width: fit-content`，让菜单撑满侧边栏宽度

第 41 行的 `<span>` 也调整：

**Before:**
```html
<span v-if="!sidebarCollapsed" class="ml-2">{{ item.label }}</span>
```

**After:**
```html
<span v-if="!sidebarCollapsed">{{ item.label }}</span>
```

原因：DaisyUI 的 `li > *` 已内置 `gap: 0.5rem`，`ml-2`（也是 0.5rem）造成双重间距。

## 验证

修改后 hover 背景应覆盖菜单项的完整宽度，与侧边栏边缘对齐（减去 DaisyUI 内置 padding）。

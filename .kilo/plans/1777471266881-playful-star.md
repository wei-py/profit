# 创建页：自动计算 & 重置按钮

## 目标

1. 输入参数值时，结果字段自动重新计算（无需手动点击"计算"）
2. 添加"重置"按钮，将输入值恢复为预设默认值

## 修改文件

### 1. `src/stores/create.js` — 添加 `resetToDefaults` 方法

在 store 中新增一个函数，将 `userInputs` 重置为当前预设的默认值，同时清空 `results` / `errors`，但保留 `basicInfo` / `images` / `variants` 和当前预设选中状态：

```js
function resetToDefaults() {
  results.value = {}
  errors.value = []
  const preset = configStore.config.presets.find(p => p.presetId === selectedPresetId.value)
  if (preset && preset.params) {
    for (const param of preset.params) {
      userInputs.value[param.fieldKey] = param.defaultValue ?? ''
    }
  }
}
```

- 暴露：在 `return` 中添加 `resetToDefaults`
- 注意：`userInputs` 是一个 `ref({})`，直接赋值属性会触发响应式更新（Vue 3 ref 已深度响应式），但 `watch` 的 `deep` 模式也需要 key-level 变化，所以用 `nextTick` 或直接重新赋值的方式确保触发。保险做法：先赋空对象再赋默认值，或者直接逐属性赋值

### 2. `src/pages/CreatePage.vue` — 自动计算 + 重置按钮

#### 2a. 添加 `watch` 实现自动计算

```js
import { computed, ref, watch } from 'vue'
```

在 script 中添加防抖 watch：

```js
let debounceTimer = null

watch(
  () => ({ ...createStore.userInputs }),
  () => {
    if (!createStore.selectedPresetId) return
    clearTimeout(debounceTimer)
    debounceTimer = setTimeout(() => {
      validationErrors.value = []
      const errs = validateUserInputs(createStore.userInputs, paramFields.value)
      if (errs.length > 0) {
        validationErrors.value = errs
        // 输入不合法时不清空之前的计算结果，但也不执行新计算
        return
      }
      createStore.calculate()
    }, 300)
  },
  { deep: true },
)
```

> 为什么用 `() => ({ ...createStore.userInputs })`：`watch` 对 `ref` 对象使用 `deep: true` 时，直接传 `createStore.userInputs` 即可。但为了确保触发，使用 `() => createStore.userInputs.value` 并展开浅拷贝确保每次变化可检测。

实际上更简洁的方式：

```js
watch(
  () => createStore.userInputs.value,
  () => {
    // debounced calculate logic
  },
  { deep: true },
)
```

但 Pinia store 的 ref 可能需要通过 `.value` 访问。在 `<script setup>` 中 destructure 出来的 `userInputs` 已经是响应式的，所以直接用 `userInputs` 即可：

```js
import { watch } from 'vue'

let calcTimer = null

watch(
  createStore.userInputs,
  () => {
    if (!createStore.selectedPresetId) return
    clearTimeout(calcTimer)
    calcTimer = setTimeout(() => {
      validationErrors.value = []
      const errs = validateUserInputs(createStore.userInputs, paramFields.value)
      if (errs.length > 0) {
        validationErrors.value = errs
        return
      }
      createStore.calculate()
    }, 400)
  },
  { deep: true },
)
```

#### 2b. 保留手动"计算"按钮

手动按钮的逻辑不变（`handleCalculate`），与自动计算并存。用户手动点击可立即触发（绕过防抖）。

#### 2c. 添加"重置"按钮

在"计算"按钮旁添加：

```html
<button class="btn btn-ghost btn-sm" @click="handleReset">
  重置
</button>
```

```js
function handleReset() {
  createStore.resetToDefaults()
  validationErrors.value = []
}
```

将重置放在 `card-actions` 中计算按钮旁边。

## 数据流变更

```
用户输入参数值
  → updateInput() 写入 store
  → watch 检测变化
  → 400ms 防抖后自动 validate + calculate
  → results 更新，右侧结果面板实时显示

用户点击"重置"
  → resetToDefaults() 将 userInputs 恢复为预设默认值
  → watch 检测变化
  → 自动重新计算
```

## 预设选择不变

`selectPreset` 的已有行为（切换预设时清空所有状态并加载默认值）保持不变。新增的 `resetToDefaults` 不会清空 `basicInfo` / `images` / `variants`。

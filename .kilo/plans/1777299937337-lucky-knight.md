# Fix: DaisyUI menu button hover display issue

## Problem
In `src/layouts/DefaultLayout.vue`, the sidebar menu's `<li><button>` items have incomplete hover highlighting due to:
1. Redundant `class="flex items-center"` overriding DaisyUI's built-in `display: grid` layout
2. `active` class should be `menu-active` per DaisyUI v5 docs (line 1162: modifier `menu-active`)

## Steps

### 1. Edit `src/layouts/DefaultLayout.vue`
- Remove `class="flex items-center"` from the `<button>` element (DaisyUI's `.menu` already handles layout)
- Change `:class="{ active: route.path === item.path }"` to `:class="{ 'menu-active': route.path === item.path }"`

**Before (line 33-42):**
```html
<button
  class="flex items-center"
  :class="{ active: route.path === item.path }"
  @click="router.push(item.path)"
>
```

**After:**
```html
<button
  :class="{ 'menu-active': route.path === item.path }"
  @click="router.push(item.path)"
>
```

### 2. Verify
Run `npm run dev` and check that hover highlights now extend full width of each menu item.

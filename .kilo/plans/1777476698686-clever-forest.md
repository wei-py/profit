# 客户端可编辑预设参数的 fieldKey

## 目标

在客户端（PresetPage）直接编辑预设参数的 `fieldKey`，无需打开 Excel 修改 `preset_params` 表。

## 根因回顾

Excel `preset_params` 表中 `field_key` 列为空 → 全部参数 `fieldKey` = `""` → 绑定到 `userInputs[""]` 同一 key → 串扰。已在 CreatePage 和 createStore 中添加过滤逻辑，但用户选该预设后看不到任何输入框，需要在 PresetPage 补充 fieldKey 值。

## 方案

### 1. 在 PresetPage 参数表中添加 `fieldKey` 可编辑列

**文件**: `src/pages/PresetPage.vue`

在现有的 `paramColumns` 数组（第 107 行）最前面插入一个 `fieldKey` 列定义：

```js
{
  key: 'fieldKey',
  prop: 'fieldKey',
  label: '字段键',
  type: 'select',
  getOptions: () => {
    const opts = configStore.config.fields.map(f => ({
      value: f.fieldKey,
      label: `${f.fieldKey}${f.fieldName ? ` (${f.fieldName})` : ''}`,
    }))
    return [{ value: '', label: '-- 未选择 --' }, ...opts]
  },
},
```

说明：
- 使用 `select` 类型，EpitableTable 会渲染为下拉选择框
- `getOptions` 从 `configStore.config.fields` 动态读取所有已定义字段
- 显示格式为 `fieldKey (fieldName)`，如 `exchange_loss (汇损)`
- 第一项为空选项，对应当前的空 fieldKey 状态

### 2. 更新 CreatePage 警告信息，引导用户到 PresetPage

**文件**: `src/pages/CreatePage.vue`

修改 `paramFields` computed 中的 `console.warn` 文案（第 47 行），增加操作指引：

```
预设中存在 fieldKey 为空的参数，已跳过。请到「预设」页面补充 field_key 值。
```

### 3. 保留 createStore 中的跳过逻辑

`src/stores/create.js` 中 `selectPreset` 和 `resetToDefaults` 的空 fieldKey 跳过 + warn 保持不变。这些是防御性代码，确保即使有遗漏的空 fieldKey 也不会导致串扰。

### 4. 移除 createStore 中的重复警告

`createStore` 的 `selectPreset`/`resetToDefaults` 中的 `console.warn` 与 CreatePage 的 warn 信息重复，保留 CreatePage 一处即可。将 store 中的 warn 改为更简洁的调试级别输出或不输出。

## 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/pages/PresetPage.vue` | `paramColumns` 数组最前面插入 `fieldKey` select 列 |
| `src/pages/CreatePage.vue` | 更新 `console.warn` 文案，添加操作指引 |
| `src/stores/create.js` | 移除 selectPreset/resetToDefaults 中重复的 console.warn（保留跳过逻辑） |

## 用户操作步骤

1. 切换到「预设」页面
2. 左侧列表选中 `preset_001`
3. 在参数表格中，每行的「字段键」列从下拉框选择合适的字段
4. 点击顶部「保存配置」按钮
5. 回到「新建」页面，重新选择该预设即可看到所有输入框

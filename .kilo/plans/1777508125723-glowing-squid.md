# 本地图床功能实现计划

## 需求分析

为 Tauri 桌面应用添加本地图床功能：支持从本地选择图片文件，将其复制到应用数据目录统一管理，并在创建页和列表页中预览展示。

## 存储位置设计

**推荐位置**：`<appDataDir>/images/`（应用数据目录下的 images 子目录）

| 平台 | 实际路径 |
|------|---------|
| macOS | `~/Library/Application Support/com.tauri.dev/images/` |
| Windows | `C:\Users\<user>\AppData\Roaming\com.tauri.dev\images\` |
| Linux | `~/.local/share/com.tauri.dev/images/` |

**理由**：
- Tauri 推荐：`appDataDir` 是 Tauri 官方指定的持久化用户数据目录
- 跨平台兼容：自动适配不同操作系统的标准数据目录
- 开发环境可用：dev 模式下同样有效，与生产行为一致
- 非捆绑资源：打包后的应用 bundle 是只读的，外部目录可自由读写

## 技术方案

### 图片处理流程

```
用户选择图片 → 读取文件内容 → 复制到 appDataDir/images/ → 存储路径到记录 → 展示时转 base64
```

### 关键 API

- `@tauri-apps/api/path` — `appDataDir()` 获取数据目录
- `@tauri-apps/plugin-dialog` — `open()` 选择图片文件
- `@tauri-apps/plugin-fs` — `readFile()` / `writeFile()` / `remove()` 文件操作
- `@tauri-apps/plugin-fs` — `mkdir()` / `exists()` 目录管理

### 展示方案

用 base64 data URL 展示本地图片（`data:image/png;base64,...`），无需修改 Tauri 安全配置。对于网络 URL 继续直接使用 `<img src="...">`。

### 路径格式

`images` 字段保持逗号分隔字符串，兼容三种来源：
- 网络 URL：`https://example.com/img.jpg`（直接用作 src）
- 本地路径：`/path/to/appData/images/uuid.png`（展示时转 base64）

## 实现步骤

### 步骤 1：创建图片管理 composable `src/composables/useImageHost.js`

核心功能：
- `initImageDir()` — 确保 `<appDataDir>/images/` 目录存在
- `pickImages()` — 调用系统文件对话框选择图片（支持多选、常用格式过滤）
- `addImages(files)` — 复制选中图片到数据目录，返回新路径列表
- `readLocalImageAsDataUrl(path)` — 读取本地图片文件并转为 base64 data URL（带缓存）
- `removeLocalImage(path)` — 删除本地图片文件
- `getImageUrls(imagesString)` — 将 images 字段字符串转为可展示的 URL 列表（URL 直接返回，本地路径转 base64）
- `clearCache()` — 清除 base64 缓存

### 步骤 2：创建图片上传组件 `src/components/common/ImageUploader.vue`

UI 功能：
- 已选图片的缩略图网格（w-20 h-20，圆角边框）
- 每张图片右上角有删除按钮
- 底部"添加图片"按钮，点击触发文件选择
- 上传中 loading 状态
- 空状态提示
- 支持 props：`modelValue`（逗号分隔字符串）、双向绑定
- emit：`update:modelValue`

### 步骤 3：修改 CreatePage.vue 图片区域

将当前 textarea + 预览替换为 ImageUploader 组件：
- 移除 `<textarea v-model="createStore.images">`
- 移除旧的 img 预览 div
- 引入并放置 `<ImageUploader v-model="createStore.images">`

### 步骤 4：修改 ListPage.vue 图片显示

更新查看/编辑弹窗中的图片渲染逻辑：
- 使用 `getImageUrls()` 解析 images 字段
- 本地路径图片转为 data URL 后显示
- 网络 URL 图片保持直接显示

### 步骤 5：后端配置调整（如需要）

检查 `tauri.conf.json` 能力权限是否需要补充 `fs` scope，当前已配置 `allow-desktop-read-recursive` 和 `allow-desktop-write-recursive` 等，可能需要补充 appDataDir 相关权限。

## 已确认

- **格式限制**：支持常见图片格式（jpg/png/gif/webp/bmp 等），不限大小
- **关联功能**：先做基础上传，变体/选项关联后续实现
- **列表页编辑**：编辑模式也使用上传组件，体验一致

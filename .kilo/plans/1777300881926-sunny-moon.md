# 项目规划：对照 readme & docs 补齐缺口

## 当前状态总结

项目骨架已完整：5 个页面、规则引擎、Excel 读写、Pinia stores、Tauri 文件 IO 均已实现。核心架构（Excel 定义规则 → 引擎解释执行 → 页面收集输入输出）与 docs 描述一致。

对照 readme 和 docs 文档，发现以下缺口：

---

## 缺口清单（按优先级）

### P0 —— 创建页缺少基础信息字段

**现状**：`CreatePage.vue` 只渲染 `preset.params` 对应的输入项，没有独立的名称、款号、成本、重量等基础信息输入区。

**readme 要求**：
> "基础信息包括名称、款号、成本、重量等。"

**修复方案**：
- 在 `CreatePage.vue` 预设选择器下方增加「基础信息」卡片
- 字段：`name`（名称）、`sku`（款号）、`cost`（成本，元）、`weight`（重量，g）
- 这些字段也应写入 `createStore.userInputs`，与 preset params 一起参与计算
- 基础信息字段可作为 `fields` sheet 中的固定字段，或者直接硬编码在创建页（因为它们是通用字段，不随规则变化）

### P0 —— 创建页缺少图片和变体字段

**现状**：`CreatePage.vue` 完全没有图片上传/输入和变体编辑功能。

**readme 要求**：
> "创建页中的字段分为基础信息、计算字段、结果字段、图片、变体"
> "图片支持上传、本地路径或链接，并可与变体和选项关联"

**修复方案**：
- 增加「图片」输入区：支持多行文本输入（逗号分隔 URL/路径），或文件选择对话框（Tauri dialog）
- 增加「变体」输入区：文本区域，JSON 格式，记录 SKU 变体信息（如颜色、尺寸变体组合）
- 图片和变体同样写入 `userInputs` 或作为 `getRecord()` 的额外字段
- 图片与变体的关联可以先用 JSON 结构表达，不急于做可视化关联编辑

### P1 —— 缺少字段管理页

**现状**：Excel 有 `fields` sheet，但 UI 中无专门页面管理字段定义。目前只能通过预设页「补全默认参数」间接创建字段，无法直接增删改字段。

**影响**：
- 新增字段需要手工编辑 Excel
- 字段的 `type`、`unit`、`optionGroupId`、`ruleMode` 等属性无处可视化编辑

**修复方案**：
- 新增 `FieldPage.vue`（路由 `/field`）
- 使用 `EditableTable` 组件管理字段
- 列：fieldKey、fieldName、type（text/number/select/boolean/date/image）、unit、optionGroupId、ruleMode（input/output/both）、sort、required
- 侧边栏增加「字段」导航项

### P1 —— 选项项缺少「备注」字段

**现状**：`OptionPage.vue` 的 `itemColumns` 只有 4 列：显示名、值、排序、启用。

**readme 要求**：
> "表格字段包括：显示名称、实际值、排序、启用、备注、操作"

**修复方案**：
- 在 `itemColumns` 中增加 `{ key: 'remark', prop: 'remark', label: '备注' }`
- Excel reader/writer 也需要同步支持 `remark` 列的读写
- `remark` 字段自始至终跟随 option item 的数据流

### P2 —— 缺少示例 `配置.xlsx`

**现状**：`docs/excel-rule-engine-example.xlsx` 存在，但位置在 `docs/` 下（不在 `public/` 中）。用户首次使用没有可直接打开的模板。

**修复方案**：
- 将 `docs/excel-rule-engine-example.xlsx` 复制到 `public/examples/` 下
- 可考虑在应用无配置时引导用户打开此示例文件

### P2 —— docs 文件夹清理

**现状**：`docs/` 下有 4 张 PNG 图（image.png ~ image4.png），可能是 Mermaid 图的截图备份，已内嵌到 md 中。

**建议**：确认这些图片是否还被引用，如无引用可清理。

### P3 —— 列表页记录字段对齐

**现状**：`getRecord()` 只合并 `userInputs + results`，不包含图片、变体等额外字段。

**修复方案**：
- `getRecord()` 补充 `images`、`variants` 等字段
- 确保创建页的图片/变体输入能流入列表记录

---

## 实现顺序

1. **P0 创建页基础信息** — 最高频使用路径，先补齐
2. **P0 创建页图片/变体** — 紧随基础信息
3. **P1 字段管理页** — 提升配置体验
4. **P1 选项备注字段** — 小改动
5. **P2 示例文件 & 清理** — 收尾

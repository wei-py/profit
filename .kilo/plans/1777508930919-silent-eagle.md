# 修复"添加图片链接"按钮无反应问题

## 问题分析

`ImageUploader.vue:54-59` 的 `handleAddUrl()` 使用 `window.prompt()` 弹窗获取 URL 输入。在 Tauri v2 的 WKWebView 环境中，`window.prompt()` 可能被静默忽略或无用户可见反馈，导致点击按钮后无任何反应。

## 解决方案

将 `window.prompt()` 替换为**内联输入框**模式：

1. 点击"添加图片链接"按钮 → 隐藏按钮，原地显示一个 input + 确认/取消按钮
2. 用户输入 URL 后点确认 → 追加到 images 列表，收起输入框
3. 用户点取消或按 Escape → 收起输入框，不做任何操作

## 实现步骤

### 修改 `src/components/common/ImageUploader.vue`

1. 新增 ref：`showUrlInput` (boolean)、`urlInput` (string)
2. 新增 `confirmUrl()` — 追加 URL 到 modelValue 并重置状态
3. 新增 `cancelUrl()` — 重置状态
4. 替换模板中"添加图片链接"按钮区域：根据 `showUrlInput` 状态切换显示按钮或内联输入框
5. 输入框支持 Enter 确认、Escape 取消
